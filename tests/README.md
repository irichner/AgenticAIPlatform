# OpenAPI Detection Test Harness

Regression harness for the MCP server builder's OpenAPI detection and tool-generation pipeline. Turns "vibes-based" confidence into trustworthy, provenance-backed numbers that PR reviewers can rely on.

## Quick start

```bash
# 1. Install backend + test dependencies
pip install -r backend/requirements.txt -r tests/requirements.txt

# 2. Download upstream specs and generate baselines (required before unit tests pass)
python scripts/refresh_fixtures.py

# 3. Run unit tests
pytest tests/unit/ -v
```

## Three tiers

| Tier | Location | When it runs | Merge-blocking? |
|---|---|---|---|
| Unit | `tests/unit/` | Every push / PR | **Yes** |
| Integration | `tests/integration/` | Opt-in (see CI) | No |
| Eval | `tests/eval/` | Nightly (opt-in) | No |

### Unit tier

Fast, fully vendored (no network, no Docker). Tests that baselines match current parser output within tolerance.

```bash
pytest tests/unit/ -v
```

### Integration tier

Requires Docker and a running mock server. Tests discovery path-probing and HTTP request shaping.

```bash
pytest tests/integration/ -v
```

### Eval tier

Requires `ANTHROPIC_API_KEY` and `OPENAI_API_KEY`. Measures recall@1 and recall@5 before and after AI description enhancement.

```bash
ANTHROPIC_API_KEY=sk-... OPENAI_API_KEY=sk-... pytest tests/eval/ -v
```

## Refreshing fixtures

When upstream specs change, run:

```bash
python scripts/refresh_fixtures.py
```

This downloads each spec, re-parses it, updates baselines in `tests/fixtures/baselines/`, and writes a drift report to `tests/fixtures/baselines/drift_report.md`.

**After refreshing:**
1. Review `drift_report.md` — verify the changes are expected
2. Update `tests/config.py` constants that are derived from baselines (add provenance comments)
3. Commit the updated baselines and config together in one PR

## Adding a synthetic fixture

Synthetic fixtures isolate a single parser behaviour in under 50 lines of JSON:

1. Create `tests/fixtures/specs/synthetic/minimal-<behaviour>.json`
2. Add a corresponding test function in `tests/unit/test_synthetic.py`
3. Keep the fixture under 50 lines — if it's longer, it's testing too many things at once

## Linode label verification

`tests/fixtures/specs/linode-labels.json` contains the held-out evaluation labels. Every `"ground_truth"` must be a real `operationId` from the Linode spec.

After refreshing fixtures, run the verification script:

```bash
python scripts/verify_linode_labels.py
```

This loads the Linode spec and checks every label. Replace any `"<verify>"` placeholders with real operationIds before running eval tests.

## Baseline numbers

Every constant in `tests/config.py` that contains a measured number has a **provenance comment** in the format:

```python
# Provenance: <how derived>, <date>, <fixture or method>
CONSTANT_NAME = value
```

Constants that haven't been measured yet use `_PendingMeasurement` and raise `NotImplementedError` on access. This is intentional — a falsely-confident threshold is worse than an explicit "not yet measured" error.

## CI

The unit tier runs on every push and PR (`.github/workflows/test.yml`). Integration and eval tiers are defined in the same workflow but disabled with `if: false`. To enable them, remove the `if: false` line from the relevant job.

## Repository layout

```
scripts/
└── refresh_fixtures.py      # Download specs, generate baselines, drift report
└── verify_linode_labels.py  # Verify linode-labels.json against actual spec

tests/
├── fixtures/
│   ├── specs/               # Vendored specs (downloaded by refresh_fixtures.py)
│   │   ├── synthetic/       # Hand-crafted minimal specs for isolated behaviour tests
│   ├── negative/            # Payloads that must NOT be recognised as OpenAPI specs
│   ├── discovery/           # FastAPI mock server for discovery tests
│   └── baselines/           # Generated by refresh_fixtures.py (commit after refresh)
├── unit/                    # Fast, vendored — merge-blocking
├── integration/             # Requires Docker + network — opt-in
├── eval/                    # Requires LLM APIs — nightly opt-in
├── conftest.py              # Shared fixtures and sys.path bootstrap
├── config.py                # Provenance-commented constants
└── requirements.txt         # Test-only dependencies
```
